Micosoft의 cloud computing인 Azure를 알게 되었고 무료 강의가 있어 공부하기로 하였다.
먼저 클라우드의 기초를 공부해보자~~~

클라우드 컴퓨팅이란
 -사용자가 직접적으로 커퓨팅 서비스를 관리하는 것이 아닌 클라루드를 통해 서비스
 (서버,스토리지,DB,네트워킹,SW 분석 및 인텔리전스 등)가 포함되어 신속하고 유연한 리소스, 규모를 제공받을 수 있다.
 -다른 사람이 운영하는 데이터 센터에서 컴퓨팅 성능 및 스토리지를 임대하는 것이다.
 -가장 큰 이점은 기본 인프라를 유지 관리해주고 비즈니스 문제를 신속히 해결 할 수 있다.

클라우드 환경
 안정성 
  선택 서비스 수준에 따라 문제 발생 시 가동 중지 시간 없이 지속적 사용자 환경을 제공 할 수 있다.
 확장성
  수직(가상 머신에서 RAM이나 CPU를 추가)
  수평(리소스의 인스턴스 추가)
 탄력성
  필요한 리소스를 항상 갖도록 구현 가능
 민첩성
  요구사항 변경 시 클라우드 기반 리소스를 신속히 배포 가능
 지리적 배포
  전 세계 지역 데이터 센터에 배포 가능
 재해 복구
 백업 서비스, 데이터 복제 및 지역 배포를 활용해 복구 가능

클라우드 서비스 모델
 IaaS
  - 가장 공급자의 관리가 적은 모델
  - 공급자 : HW(하드웨어)를 최신 상태로 유지 (Azure VM)
    테넌트 : OS 유지 관리 및 네트워크 구성은 사용자가 책임을 가진다
  가능 서비스
   - 서버 및 스토리지
   - 네트워킹 방화벽/보안
   - 데이터 센터 물리적 공장/건물
PaaS
  - "관리형 호스팅 환경"
  - 공급자 : 가상 머신 및 네트웨킹 리소스를 관리
    테넌트 : 애플리케이션을 관리형 호스팅 환경에 배포
  - App Services
     개발자가 실제 HW/SW 요구 사항 처리 없이 어플리케이션을 업로드 할 수 있는 관리형 호스팅 환경 제공
  -가능 서비스 
   - IaaS + OS ,개발도구(DB관리, 비즈니스 분석)
SaaS
  - 공급자 : 가상 머신,네트워킹 리소스, 데이터 스토리지, 어플리케이션 등 모든 애플리케이션 환경 측명을 관리
  - 테넌트 : 공급자가 관리하는 애플리케이션에 data만 제공
  -가능 서비스 
   - PaaS + 호스팅된 애플리케이션/웹

서버리스 컴퓨팅
 - PaaS와 겹치는 서버리스 컴퓨팅을 사용하면 개발자가 인프라 관리 없지 빠르게 애플리케이션을 빌드 할 수 있다.
 - 공급자 : 코드 실행에 필요한 인프라를 자동으로 프로비저닝, 스케일링 및 관리
 - 확장성이 뛰어나며 특정함수, 트리거 방생 시 리소스를 함
  주의점
   - 서버에서 계속 코드를 실행 하고 있음
   - 프로비저닝 및 관리와 관련된 작업이 개발자에게 표시 x

세가지 배포 모델
 - 퍼블릭 클라우드
    공용 인터넷을 통해 제공, 서비스 구매 사용자에게 모두 제공
 - 프라이빗 클라우드
    컴퓨팅 리소스는 단일 기업 또는 조직의 사용자가 독점으로 사용
    온-사이트 데이터 센서에 물리적으로 매치 가능하고 다사 서비스 공급자에 의해 호스트 가능함.
 - 하이브리드 클라우드
    두 클라우드 간에 데이터 및 애플리케이션 공유 가능
※
클라우드 공급자 : 클라우드를 공급하는 사람
클라우드 테넌트 : 클라우드를 구매한 유저
프로비저닝 : 사용자의 요구에 맞게 시스템 자원을 미리 준비 해두는 것
