최단 경로에 대해 학습하기!
 최단 경로는 말그대로 가장 짧은 경로, 즉 "길 찾기" 문제라고 불리는 알고리즘이다.
 대표적으로 한지점에서 다른 특정 지점, 모든지점에서 모든지점의 최단경로를 구하는 문제가 많다.
 표시
  그래프를 이용하여 표현한다
  graph[index]=value ---> index:노드 value: 간선

 최단 거리 알고리즘 
  1.다익스트라 알고리즘
    특정 노드에서 출발하여 다른 노드로 가는 각각의 최단 경로를 구하는 알고리즘
    ※음의 간선이 없을 때 정상 작동한다.
   주로 GPS 소프트웨어의 기본 알고리즘으로 사용한다.
   실행 단계
   -출발 노드 설정
   -최단 거리 테이블(1차원 리스트) 초기화
   -방문하지 않은 노드 중에 최단거리가 짧은 노드를 선택
   -해당 노드를 거쳐 다른 노드로 가는 비용을 계산하여 최단 거리 테이블 갱신
   - 3,4번과정을 반복
   ※초기상태의 모든 노드로 가능 최단거리는 무한으로 설정한다---> int(1e9) (10억이기 때문에 설정)
   ※일반적으로 다익스트라 알고리즘은 ☆그리드 알고리즘이다☆  
   ★한단계당 하나의 노드에 대한 최단 거리를 확실히 찾는것
   ★★★heapq 라이브러리 활용 잘하기!!!
    
 2.플로이드 워셜 알고리즘
    "모든지점에서 다른 모든 지점까지의 최단 경로를 모두 구해야 하는 경우"
    매번 방문하지 않는 노드 중 최단 거리를 갖는 노드를 찾을 필요가 없음
    n번의 단계가 수행하며 단계마다O(N^2)의 연산을 통해 현재 노드를 거처가는 모든 경로를 고려한다.
    따라서 O(N^3)의 시간복잡도가 존재한다.
    ※일반적으로 플로이드 워셜 알고리즘은 ☆다이나믹 알고리즘이다☆
   점화식
     D(a,b) = min(D(a,b),D(a,k)+D(k,b))
   실행단계
   -n,m 노드 개수,간선 개수 입력
   -최단 거리 테이블(2차원 리스트) 초기화
   -if a==b : D(a,b)=0 초기화 자기 자신 최단경로
   - 간선입력
   -3중 for문을 사용하여 점화식 수행
